# 卡牌战争 (Card War) - 游戏设计文档

## 游戏概述

一款独特的**卡牌对垒策略游戏**，双方各控制一支由11个单位组成的军队，通过层层传递战力单元来攻击敌方大营。

---

## 核心机制

### 1. 阵型结构

双方各有 **4 排** 阵型，共 **11 个单位位置**：

```
玩家A视角：
┌─────────────────────────────────────┐
│  [A-大营]                           │ ← 只有1个单位，核心
│  [A-先锋1][A-先锋2][A-先锋3]        │ ← 3个单位
│  [A-中军1][A-中军2][A-中军3]        │ ← 3个单位
│  [A-殿后1][A-殿后2][A-殿后3]        │ ← 3个单位
└─────────────────────────────────────┘

战场对垒布局（交错排列）：
┌──────────────────────────────────────────────┐
│  [A-大营]                                    │
│  [B-先锋1][B-先锋2][B-先锋3]                │
│  [A-殿后1][A-殿后2][A-殿后3]                │
│  [B-中军1][B-中军2][B-中军3]                │
│  [A-中军1][A-中军2][A-中军3]                │
│  [B-殿后1][B-殿后2][B-殿后3]                │
│  [A-先锋1][A-先锋2][A-先锋3]                │
│  [B-大营]                                    │
└──────────────────────────────────────────────┘
```

### 2. 单位类型

| 排 | 单位数量 | 功能 |
|----|---------|------|
| **大营** | 1 | 核心，有生命值，被攻破则失败 |
| **先锋** | 3 | 前线进攻单位，负责最终输出伤害 |
| **中军** | 3 | 中间传递，接收殿后传递的战力 |
| **殿后** | 3 | 后方支援，接收大营派遣的战力单元 |

### 3. 战力单元 (Power Unit)

- 大营每个回合生成一定数量的**战力单元**
- 战力单元是可以传递的"能量"
- 战力单元最终转化为对敌方大营的伤害

### 4. 回合流程

双方**交替进行进攻/防守回合**。

#### 进攻方回合流程：

```
大营生成战力
    ↓
派遣战力到殿后（分配）
    ↓
殿后传递到中军
    ↓
中军传递到先锋
    ↓
先锋攻击敌方大营（造成伤害）
```

具体步骤：

1. **生成阶段**：大营生成基础战力（如 5 点）
2. **派遣阶段**：决定将多少战力派遣到哪个殿后单位
3. **传递阶段**：殿后 → 中军 → 先锋，自动逐层传递
4. **攻击阶段**：先锋的战力转化为对敌方大营的伤害

#### 防守方回合：

- 可以进行防御部署
- 可以使用卡牌技能
- 准备下一回合的战力分配

## 卡牌系统

### 卡牌类型

1. **单位卡**：部署到特定位置，增强该位置的能力
2. **战术卡**：一次性效果，如增援、突袭、防守等
3. **计谋卡**：特殊效果，如截断敌方传递、双倍传递等

### 卡牌效果示例

| 卡牌名称 | 类型 | 效果 |
|---------|------|------|
| 精锐先锋 | 单位 | 部署到先锋，攻击力+2 |
| 紧急增援 | 战术 | 大营立即生成3点额外战力 |
| 截断补给 | 计谋 | 敌方本回合传递效率-50% |
| 连环传递 | 计谋 | 本回合传递效率+50% |

---

## 战斗计算

### 基础伤害公式

```
实际伤害 = 先锋战力 × 先锋攻击系数 - 敌方大营防御
```

### 传递效率

- 默认传递效率：100%
- 部分卡牌会影响传递效率
- 若单位位置为空，战力会损失一定比例

### 胜利条件

- 敌方大营生命值归零 → 胜利
- 己方大营生命值归零 → 失败

---

## 游戏流程

1. **准备阶段**：双方布置初始单位卡
2. **交替回合**：轮流进行进攻/防守
3. **胜利判定**：一方大营生命值为0时结束

---

## 策略要点

1. **资源分配**：决定将战力集中到哪个殿后单位
2. **阵型布置**：合理部署单位卡增强关键位置
3. **时机把握**：使用战术卡打断敌方或增强己方
4. **传递链管理**：确保传递链畅通，避免战力损失

---

## 技术实现要点

1. **阵型数据结构**：11个位置的数组/列表
2. **战力传递系统**：递归或迭代传递算法
3. **回合状态机**：生成→派遣→传递→攻击
4. **卡牌效果系统**：可扩展的效果触发机制
